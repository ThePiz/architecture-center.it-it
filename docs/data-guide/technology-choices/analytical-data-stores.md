<span data-ttu-id="ea98f-103">In un'architettura per [Big Data](../big-data/index.md) è spesso necessario un archivio dati analitici in grado di servire dati elaborati in un formato strutturato su cui è possibile eseguire query con strumenti di analisi.</span><span class="sxs-lookup"><span data-stu-id="ea98f-103">In a [big data](../big-data/index.md) architecture, there is often a need for an analytical data store that serves processed data in a structured format that can be queried using analytical tools.</span></span> <span data-ttu-id="ea98f-104">Gli archivi dati analitici che supportano l'esecuzione di query per ottenere dati dal percorso ad accesso frequente e da quello ad accesso sporadico sono collettivamente denominati livello di gestione o archiviazione di gestione dati.</span><span class="sxs-lookup"><span data-stu-id="ea98f-104">Analytical data stores that support querying of both hot-path and cold-path data are collectively referred to as the serving layer, or data serving storage.</span></span>

<span data-ttu-id="ea98f-105">Al livello di gestione vengono trattati i dati elaborati ottenuti da entrambi i percorsi.</span><span class="sxs-lookup"><span data-stu-id="ea98f-105">The serving layer deals with processed data from both the hot path and cold path.</span></span> <span data-ttu-id="ea98f-106">Nell'[architettura lambda](../big-data/index.md#lambda-architecture) il livello di gestione
