## <a name="high-availability-and-failover"></a><span data-ttu-id="8aeb8-123">Disponibilità elevata e failover</span><span class="sxs-lookup"><span data-stu-id="8aeb8-123">High availability and failover</span></span>

## <a name="scalability"></a><span data-ttu-id="8aeb8-132">Scalabilità</span><span class="sxs-lookup"><span data-stu-id="8aeb8-132">Scalability</span></span>

<span data-ttu-id="8aeb8-133">I mainframe supportano in genere la scalabilità verticale, mentre gli ambienti cloud supportano lo scale-out. <span data-ttu-id="8aeb8-134">Inoltre, una struttura CF offre risorse di calcolo ad accoppiamento stretto, mentre le funzionalità di scale-out di Azure sono ad accoppiamento lasco.</span><span class="sxs-lookup"><span data-stu-id="8aeb8-134">In addition, a CF offers tightly coupled compute, whereas the scale-out features of Azure are loosely coupled.</span></span> <span data-ttu-id="8aeb8-135">Il cloud può essere ampliato o ridotto per soddisfare esattamente le specifiche dell'utente, con potenza del computer, risorse di archiviazione e servizi ridimensionabili su richiesta con un modello di fatturazione basato sull'utilizzo.</span><span class="sxs-lookup"><span data-stu-id="8aeb8-135">The cloud can scale up or down to match exact user specifications, with compute power, storage, and services scaling on demand under a usage-based billing model.</span></span>
