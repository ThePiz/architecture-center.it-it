# <a name="materialized-view-pattern"></a><span data-ttu-id="051f4-104">Modello di vista materializzata</span><span class="sxs-lookup"><span data-stu-id="051f4-104">Materialized View pattern</span></span>

<span data-ttu-id="051f4-105">Generare viste prepopolate sui dati in uno o più archivi dati quando i dati non sono formattati in modo ideale per le operazioni di query necessarie.</span><span class="sxs-lookup"><span data-stu-id="051f4-105">Generate prepopulated views over the data in one or more data stores when the data isn't ideally formatted for required query operations.</span></span> <span data-ttu-id="051f4-106">In questo modo è possibile garantire l'efficienza delle attività di query ed estrazione dei dati e migliorare le prestazioni dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="051f4-106">This can help support efficient querying and data extraction, and improve application performance.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="051f4-107">Contesto e problema</span><span class="sxs-lookup"><span data-stu-id="051f4-107">Context and problem</span></span>

<span data-ttu-id="051f4-108">Quando si archiviano i dati, la priorità per gli sviluppatori e gli amministratori dei dati è spesso rappresentata dalla modalità di archiviazione dei dati, piuttosto che da quella di lettura.</span><span class="sxs-lookup"><span data-stu-id="051f4-108">When storing data, the priority for developers and data administrators is often focused on how the data is stored, as opposed to how it's read.</span></span> <span data-ttu-id="051f4-109">Il formato di archiviazione scelto è in genere strettamente correlato al formato dei dati, ai requisiti per la gestione delle dimensioni e dell'integrità dei dati e al tipo di archivio in uso.</span><span class="sxs-lookup"><span data-stu-id="051f4-109">The chosen storage format is usually closely related to the format of the data, requirements for managing data size and data integrity, and the kind of store in use.</span></span> <span data-ttu-id="051f4-110">Quando, ad esempio, si usa l'archivio documenti NoSQL, i dati sono spesso rappresentati come una serie di aggregati, ognuno contenente tutte le informazioni per l'entità.</span><span class="sxs-lookup"><span data-stu-id="051f4-110">For example, when using NoSQL document store, the data is often represented as a series of aggregates, each containing all of the information for that entity.</span></span>

<span data-ttu-id="051f4-111">Ciò, tuttavia, può avere un effetto negativo sulle query.</span><span class="sxs-lookup"><span data-stu-id="051f4-111">However, this can have a negative effect on queries.</span></span> <span data-ttu-id="051f4-112">Quando una query richiede solo un subset dei dati di alcune entità, ad esempio un riepilogo degli ordini per diversi clienti senza tutti i dettagli degli ordini, è necessario estrarre tutti i dati per le entità rilevanti per ottenere le informazioni necessarie.</span><span class="sxs-lookup"><span data-stu-id="051f4-112">When a query only needs a subset of the data from some entities, such as a summary of orders for several customers without all of the order details, it must extract all of the data for the relevant entities in order to obtain the required information.</span></span>

## <a name="solution"></a><span data-ttu-id="051f4-113">Soluzione</span><span class="sxs-lookup"><span data-stu-id="051f4-113">Solution</span></span>

<span data-ttu-id="051f4-114">Per consentire query efficaci, una soluzione comune consiste nel generare in anticipo una vista che materializza i dati in un formato adattato al set di risultati richiesto.</span><span class="sxs-lookup"><span data-stu-id="051f4-114">To support efficient querying, a common solution is to generate, in advance, a view that materializes the data in a format suited to the required results set.</span></span> <span data-ttu-id="051f4-115">Il modello di vista materializzata descrive la generazione di viste prepopolate dei dati in ambienti in cui i dati di origine non sono in un formato adatto per l'esecuzione di query, in cui è difficile generare una query appropriata o in cui le prestazioni di query sono scarse a causa della natura dei dati o dell'archivio dati.</span><span class="sxs-lookup"><span data-stu-id="051f4-115">The Materialized View pattern describes generating prepopulated views of data in environments where the source data isn't in a suitable format for querying, where generating a suitable query is difficult, or where query performance is poor due to the nature of the data or the data store.</span></span>

<span data-ttu-id="051f4-116">Queste viste materializzate, che contengono solo i dati richiesti da una query, consentono alle applicazioni di ottenere rapidamente le informazioni necessarie.</span><span class="sxs-lookup"><span data-stu-id="051f4-116">These materialized views, which only contain data required by a query, allow applications to quickly obtain the information they need.</span></span> <span data-ttu-id="051f4-117">Oltre a unire le tabelle o combinare le entità di dati, le viste materializzate possono includere i valori correnti delle colonne calcolate o degli elementi di dati, i risultati della combinazione di valori o dell'esecuzione di trasformazioni sugli elementi di dati e i valori specificati come parte della query.</span><span class="sxs-lookup"><span data-stu-id="051f4-117">In addition to joining tables or combining data entities, materialized views can include the current values of calculated columns or data items, the results of combining values or executing transformations on the data items, and values specified as part of the query.</span></span> <span data-ttu-id="051f4-118">Una vista materializzata può anche essere ottimizzata per una singola query.</span><span class="sxs-lookup"><span data-stu-id="051f4-118">A materialized view can even be optimized for just a single query.</span></span>

<span data-ttu-id="051f4-119">Un punto importante è che una vista materializzata e i dati in essa contenuti possono essere eliminati completamente, perché è possibile ricreare interamente la vista dagli archivi dati di origine.</span><span class="sxs-lookup"><span data-stu-id="051f4-119">A key point is that a materialized view and the data it contains is completely disposable because it can be entirely rebuilt from the source data stores.</span></span> <span data-ttu-id="051f4-120">Una vista materializzata non viene mai aggiornata direttamente da un'applicazione ed è quindi una cache specializzata.</span><span class="sxs-lookup"><span data-stu-id="051f4-120">A materialized view is never updated directly by an application, and so it's a specialized cache.</span></span>

<span data-ttu-id="051f4-121">Quando i dati di origine per la vista cambiano, la vista deve essere aggiornata per includere le nuove informazioni.</span><span class="sxs-lookup"><span data-stu-id="051f4-121">When the source data for the view changes, the view must be updated to include the new information.</span></span> <span data-ttu-id="051f4-122">È possibile pianificare questa operazione in modo che avvenga automaticamente oppure quando il sistema rileva una modifica ai dati originali.</span><span class="sxs-lookup"><span data-stu-id="051f4-122">You can schedule this to happen automatically, or when the system detects a change to the original data.</span></span> <span data-ttu-id="051f4-123">In alcuni casi potrebbe essere necessario rigenerare manualmente la vista.</span><span class="sxs-lookup"><span data-stu-id="051f4-123">In some cases it might be necessary to regenerate the view manually.</span></span> <span data-ttu-id="051f4-124">La figura mostra un esempio di come è possibile usare il modello di vista materializzata.</span><span class="sxs-lookup"><span data-stu-id="051f4-124">The figure shows an example of how the Materialized View pattern might be used.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="051f4-126">Considerazioni e problemi</span><span class="sxs-lookup"><span data-stu-id="051f4-126">Issues and considerations</span></span>

<span data-ttu-id="051f4-127">Prima di decidere come implementare questo modello, considerare quanto segue:</span><span class="sxs-lookup"><span data-stu-id="051f4-127">Consider the following points when deciding how to implement this pattern:</span></span>

<span data-ttu-id="051f4-128">Come e quando la vista verrà aggiornata.</span><span class="sxs-lookup"><span data-stu-id="051f4-128">How and when the view will be updated.</span></span> <span data-ttu-id="051f4-129">Idealmente, verrà rigenerata in risposta a un evento che indica una modifica ai dati di origine, anche se ciò può causare un sovraccarico eccessivo se i dati di origine cambiano rapidamente.</span><span class="sxs-lookup"><span data-stu-id="051f4-129">Ideally it'll regenerate in response to an event indicating a change to the source data, although this can lead to excessive overhead if the source data changes rapidly.</span></span> <span data-ttu-id="051f4-130">In alternativa, è possibile usare un'attività pianificata, un trigger esterno o un'azione manuale per rigenerare la vista.</span><span class="sxs-lookup"><span data-stu-id="051f4-130">Alternatively, consider using a scheduled task, an external trigger, or a manual action to regenerate the view.</span></span>
